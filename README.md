# INTRA - Static JSON Architecture

Этот проект переведен на полностью статическую архитектуру с использованием JSON в качестве базы данных. Это позволяет деплоить сайт как обычную статику, сохраняя при этом возможность удобного редактирования данных в режиме разработки.

## 📁 Структура проекта

```
intra-v2/
├── server/
│   ├── static-cms.js    # CMS-сервер для записи данных (только для DEV)
│   └── index.js         # Хостинг статики для Production (опционально)
├── public/
│   ├── projects.json    # База данных проектов
│   ├── team-members.json # База данных команды
│   └── projects-images/ # Локальное хранилище изображений
├── src/                 # Исходный код Vue.js
└── dist/                # Готовый статический сайт (после build)
```

---

## 🚀 Быстрый старт (Разработка)

### 1. Установка
```bash
npm install
```

### 2. Запуск проекта
```bash
npm run start:all
```
Эта команда запустит:
*   **Vite** (Frontend) на порту 5173
*   **CMS Server** (Backend для записи JSON) на порту 3005

### 3. Наполнение контентом
Перейдите в раздел **Админка** (`/admin/projects`). Теперь вы можете:
1. Добавлять/редактировать проекты.
2. Загружать фото (они автоматически сохранятся в `public/projects-images/`).
3. При сохранении данные обновят `public/projects.json`.

---

## 🌍 Деплой на Production

### 1. Сборка
```bash
npm run build
```
После сборки все ваши данные из JSON и картинки будут упакованы в папку `dist`.

### 2. Размещение
Вы можете просто залить содержимое папки `dist` на любой статический хостинг (GitHub Pages, Netlify, Vercel, ahost и т.д.). **Сервер и база данных для работы сайта в продакшене не нужны.**

---

## 🔐 Безопасность (ngrok)

Если вы используете **ngrok** для удаленного доступа:
1. Настройте `CMS_SECRET_TOKEN` в файле `.env`.
2. Фронтенд автоматически проксирует запросы к CMS через основной туннель.
3. Доступ к записи данных защищен токеном.

---

## ✅ Основные преимущества
*   **Zero-cost hosting:** Не нужна оплата БД и Node.js сервера в продакшене.
*   **Speed:** Сайт грузится мгновенно как статика.
*   **Git-friendly:** Все изменения контента сохраняются в истории Git.
